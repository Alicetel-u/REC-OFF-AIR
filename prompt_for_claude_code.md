# Claude Code 用 : YouTuber実況ホラー「Ghost Streamer (仮)」開発指示

## 背景
Godot 4.x を使用して、プレイヤーがYouTuberとして心霊スポットを探索・配信するホラーゲームのプロトタイプを作成したい。シナリオよりも先に、まずゲームとして「動く部分」を優先的に実装する。

## 実装要件

### 1. プレイヤーシステム (First-Person)
*   カメラ（ビデオカメラ視点）を介した一人称操作。
*   WASD移動、Shiftでダッシュ。
*   カメラの揺れ（ボブ機能）と滑らかな視点操作。
*   懐中電灯（トグル式）の実装。

### 2. ビデオカメラUI & 演出
*   **Overlay**: 画面端に「REC」点滅、バッテリー残量、タイムコードを表示。
*   **Visual FX**: シェーダーを使用して、VHS風のノイズや走査線、わずかな画面の歪みを適用。

### 3. 疑似配信システム (Chat System)
*   **Chat UI**: 画面の一部にリアルタイムでコメントが流れるスクロール欄を作成。
*   **Logic**: 
    - プレイヤーの特定の行動（移動開始、懐中電灯点灯、特定のオブジェクトを見る）をトリガーに、ランダムなコメントを生成。
    - 視聴者数（Viewers）のカウントが時間経過やイベントで増減する仕組み。

### 4. ステージプロトタイプ
*   CSG (Constructive Solid Geometry) ノードを使用して、まず歩き回れるシンプルな「廃墟や通路」のテストマップを作成。

## 技術スタック
*   Godot 4.x (GDScript)
*   3D (Forward+ or Mobile renderer)

## 最初の一歩として依頼したいこと
「まずはGodotプロジェクトの初期フォルダ構造を作成し、基本的なプレイヤー移動とカメラUIのプロトタイプを組んでください。」
